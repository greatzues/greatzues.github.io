react-native升级

升级react-native引发的一系列bug

我不是标题党啊！我新建文件的时候偷懒就没有把版本输进去。然后记录这篇博客。

因为要使用到多张图片上传，所以在github里面输入关键字multiple images picker，找到最多star的库，我也想过自己写的，但是考虑到编写和测试的时间成本，果断放弃了。

使用ivpusic/react-native-image-crop-picker的时候出了一个bug，当时没有截图，是说需要的react-native版本在0.32.0以上，我的版本还在0.30.0，npm install会出现警告。

于是我开始备份了项目然后去升级版本，首先是去react-native中文网上查找升级的步骤

使用npm info react-native查看最新的依赖包版本。然后在项目目录下打开package.json文件，找到react-native修改到最新版本号，我这里是0.35.0.最后使用npm install开始更新。
开始警告，说我的react版本也应该更新，然后重新使用npm info react-native找到对应的版本号为15.3.1。ok，继续修改一下react版本，ok。这样就算更新好了。

开始使用react-native run-android跑原来程序，第一次跑失败了，因为原来的react-native-blur库，也就是用来是实现毛玻璃效果的库说打开不了扩展文件..\node_modules\react-native-blur\android\build\outputs\aar\react-native-blur-release.aar，于是我又开始了fix bug之旅。

在Could not expand ZIP关键词情况下我找到了Mystra的博客文章，里面说只需要删掉原来的重新下载就可以了，果然npm uninstall react-native-blur之后再重新install就没报这个错了。

可是随之而来的是另外一个bug，那就是当我使用react-native-image-crop-picker里面的命令时，当我输入npm i react-native-image-crop-picker --save这条命令时开始报这个错了

failed to find Build Tools revision 23.0.3 reactnative
根据google拿到的解决办法，我打开了android SDK将android SDK build-tools 23.0.3给勾上了。再次跑一次程序，又出现bug了。

这次是这个bug,The error is "Error: Width (0) and height (0) cannot be < = 0,这个bug在uCrop里面很多人都说可以参考这里的方法来解决Using v1.4.1, could not build because of an error. #84
由于我比较笨，怎么试都没有试成功。我也按照里面说的

or this to Gradle Plugin 1.5

我的gradle是1.5版本的，所以我一直都试下面那个方法，可是一样还是报错。后面无意中想到，要不更新一下gradle试一下上面一个的方法，然后就借鉴简书的这篇博客【Android】升级Gradle版本（Android Studio）。
按照里面的说法我用androidstudio打开了我的项目，将目录gradle\wrapper\gradle-wrapper.properties里的这个文件由原来的2.4.0修改成2.10.0,接着sync一下，重新编译项目，就跑起来了。
