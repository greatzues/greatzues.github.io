我是时间线2017/0724，我来添加一点关于git使用的新知识

在开发设计联盟的时候，当时是新建了一个分支zues在仓库上，也就是除了master之外，还有另外一个分支，而我开发主要是在zues分支上，由于项目赶得紧，所以陶醉在分支上一直没有合并。

这里需要提醒我自己的是合并的三部曲，配上每一条指令的含义，虽然这很基础啊，但我记住真的不好（其实是很少用这些指令，平时都是用commit、push、pull），所以记录下来防止下次忘记了找不到笔记看

克隆分支仓库三部曲：

  • git clone <仓库地址> ，这个我就不解释了
  • git checkout origin/zues , 切换到远程仓库的zues分支上，这时候就把对应zues仓库的代码同步到本地了
  • git checkout -b zues , 本地新建了一个zues分支用来对应远程仓库的zues分支
  • git checkout master , 切换会本地master分支
  • git merge zues , 这一步是关键，用来合并本地zues，如果这时候代码之间有冲突的话就要继续解决冲突了，但由于我是直接基于zues分支开发，没有并行开发，所以就没有解决冲突这一步
  • git branch -d zues , 删除本地zues分支
  • git branch ,查看当前分支情况，可以发现当前分支只剩下master了
  • git commit -m “”, commit代码到工作区
  • git push origin master ,推送代码到github

ok，就几个步骤，完成代码本地合并问题，希望后面我用不上这里的笔记，可以通过这次记录永久记忆下来，哈哈！
