记ios登录功能

2017-04-26 • 标签: ios

入门ios一个星期，开始写ios，对于Xcode有点懵逼，我要写一篇博客来理清楚整个流程的顺序，为后面的编写加印象

前提

封装好网路请求库，创建model、controller、Operation事件处理三者的父类，然后开始干活

创建model

继承父类BaseModel，根据和服务器说好的协议，也就是根据返回的json来创建变量，通过传入有网络请求拿到的数据，用NSDictionary字典将其保存起来并且传入到infoFromDict函数，返回一个UserModel对象，完成整个model层的创建。

UserModel.m

创建Controller

Controller类与view类关联，处理的是对应的username，password的获取，登录按钮的操作，以后还包括将数据呈现在view上。

其中，处理数据的类就交给了Operation代理类，在Operation类中将请求上传到服务器，并且将返回的数据拿到。

创建DALogin数据代理类

在父类中请求数据，加载数据，解析数据，处理成功或者失败之后的操作，DALogin只需要进行重写解析成功之后的操作即可。

上面的代码显示了，将请求拿到数据，传入到model类，通过model类转化为本地的json数据，可以填充到view上，这里用来告知用户登录成功，也就是这里的opSuccess函数

通过opSuccess函数，我们可以把传进来的info，也就是服务器返回的数据展示到view上，通过alert等形式告诉用户登录成功了，也可以在这里执行页面跳转的操作。

创建和展示View

新建一个view，也就是这里的LoginPage.xib，通过拖拽和配置图片的方式将需要的控件拼接进去

然后将控件调整到合适的位置，我这里其实有点疑惑，因为这里是通过在界面使用某一种iphone设备来调试，也就是说，我觉得是绝对定位，主要是控件的长度宽度到时遇到适配就可能很蛋疼了。

这里来介绍一下关于最左边的六个顶部按钮的作用

  • show the file inspector译为“显示文件检查器”，可以查看view文件名字、类型、相对位置、绝对路径、界面构建器文档、本地化
  • show Quick help inspector译为“显示帮助检查器”，有声明和描述，反正是一大段英文
  • show the indentity inspector译为“显示身份检查器”，可以定制一个类和模块，暂时还不知道怎么用
  • show the attribute inspector译为“显示属性检查器”，可以显示对应选中控件的所有属性，例如Button有button type有Custom、system、Detail Disclosure、info Light、info Dark，还有颜色等等
  • show the size inspector译为“显示尺寸检查器”，定制组件的大小和位置
  • show the connection inspector译为“显示连接检查器”，可以将对应组件在这里与对应的代码相关联

很多具体的用法还不知道，这里只是为了加深印象，理清后面要写的思路。

最后通过点击Placeholders还有view来完成对应的关联操作，关联也相当简单，只需要像连线一样，把控件连接起来就可以了。

预编译头文件

为了更方便在全局使用一些需要经常要用到的类，可以在Xcode中创建一个pch的预编译头文件，然后在文件中添加的类就可以不需要导入，都可以使用了，超级方便。

步骤

然后在项目的Build Settings中找到Apple LLVM 8.0 - Language
选中Prefix Header并且将pch文件路径写进去
