三种命名空间

作用：我的理解是标识属性的命名，解决命名上的冲突。

1和2命名空间里的属性是系统封装好的，第3种命名空间里的属性是用户自定义的

对于第三种，自定义的xmlns的来源其实是自定义view里面的自己定义的属性，而那个app只是一个名字，可以把xmlns:app改成xmlns:test,效果都是一样
举一个比较典型的例子，我们现在都在用的一个android.support.design.widget.CoordinatorLayout,它属于一个自定义的viewGroud，它有自己专属的属性，例如layout_behavior属性，可以用来控制子组件的滚动效果，但是安卓预定义没有这个属性名字，那么我们用来表示标识的时候就要用到自己的自定义的命名空间了，所以才出现了xmlns:app=”…”，可以用来区分属性。

java的protect关键字

一直被这个关键困扰着，本来以为自己已经熟练掌握了，但是才发现，protect还没理解好，关于public和private两个极端就不说了

  • priavte 本类可见
  • public 所有类可见
  • protected 本包和所有子类都可见
  • friendly 本包可见（即默认的形式）
    也就是protected：protected对于子女、朋友来说，就是public的，可以自由使用，没有任何限制，而对于其他的外部class，protected就变成private。

java的assert关键字

assert关键字语法很简单，有两种用法：

1、assert
如果为true，则程序继续执行。
如果为false，则程序抛出AssertionError，并终止执行。

2、assert : <错误信息表达式>
如果为true，则程序继续执行。
如果为false，则程序抛出java.lang.AssertionError，并输入<错误信息表达式>。

使用assert断言有陷阱，详情可以看这篇博客[Java陷阱之assert关键字详解](http://m.jb51.net/article/41155.htm)

adapter简单使用

常用的有 BaseAdapter，SimpleAdapter，ArrayAdapter，SimpleCursorAdapter
1、ArrayAdapter是最简单的adapter了，只显示纯文本，ArrayAdapter(Context context, int textViewResourceId, List<T> objects)的第一个参数是上下文，一般都是this；第二个参数比如android.R.layout.simple_list_item_1，就是每一个text的排列方式，第三个就是数据列表了

2、SimpleAdapter的扩展性最好,可以定义各种不同的item view布局，SimpleAdapter adapter = new SimpleAdapter(this, getData(), R.layout.simple, new String[] { "title", "img" }, new int[] { R.id.title, R.id.img });
同样，this代表上下文，getData是要适配进去的数据，第三个是每一个item view的xml文件，接着就是数据中的不同数组对象的键，通过键可以读取对应的值，放到第五个参数对应的child view中，第五个是item view不同的child view id;

3、SimpleCursorAdapter，这个我没用过，不过也很好理解，一定要以数据库作为数据源的时候,才能使用SimpleCursorAdapter

要在AndroidManifest.xml文件中加入权限，<uses-permission android:name="android.permission.READ_CONTACTS"></uses-permission>

区别final变量、final方法、final类

  • fianl变量，一般和static关键字一起使用声明一个常量,这样可以保证共享内存，最关键的是，final变量是只读的
  • final方法，如果一个方法的功能已经很完整了，子类不需要改变，可以声明为final方法，值得注意的是，final方法比非final方法更快，因为在编译的时候已经静态绑定，不需要在运行时动态绑定
  • final类的功能是完整的，不能被继承， 例如String、Interger

final有几个优点

  • final关键字提高了性能。JVM和Java应用都会缓存final变量。
  • final变量可以安全的在多线程环境下进行共享，而不需要额外的同步开销。
  • 使用final关键字，JVM会对方法、变量及类进行优化。

Application类的用法

定义

通过单例创造整个应用中唯一的Application类，生命周期就是应用的生命周期，应用内所有的activity都运行在该application所在的进程上。
因为所有组件都可以使用getApplication来获取application对象，而且这个对象都是相同的，所有application也可以进行数据共享、数据缓存

使用方法

安卓应用创建的时候会自定义一个application，作为该程序的入口，可以通过继承Application来自定义自己的application对象，getApplication()这个方法可以用来获取applicaltion对象。

通过继承Application类来实现应用程序级的全局变量，这种全局变量方法相对静态类更有保障，直到应用的所有Activity全部被destory掉之后才会被释放掉

SharedPreferences对象

在整个应用中，可以通过SharedPreferences来保存对象，实现方式有很多，我陈列几种：

  • Context.getSharedPreferences()，获取的数据可以被同一应用
  • Activity.getPreferences()，数据只能在该Activity中共享
  • PreferenceManager.getDefaultSharedPreferences()，与getSharedPreferences()不同的是，这个使用默认名称，例如：com.example.something_prefer

操作模式

  • Context.MOED_PRIVATE，默认操作模式，只能在应用本身访问，写入的内容会覆盖原文件的内容
  • Context.MOED_APPEND，检查文件是否存在，存在则追加内容，否则就新创建
  • Context.MOED_WORLD_READABLE，表示当前文件可以被其它应用阅读
  • Context.MODE_WORLD_WRITAEBLE，表示当前文件可以被其它应用写入
